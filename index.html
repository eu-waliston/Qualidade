Riscos em Projetos de Software

Identificação, Análise e Mitigação

Projeto Acadêmico - Yriane de Morais & Waliston Euripedes

Universidade Federal de Santa Maria [UFSM] - [2 Set 2025]

Resumo Executivo

Este projeto acadêmico explora a gestão de riscos em projetos de software, abordando desde a identificação até a mitigação dos principais riscos. Através de uma revisão teórica e um estudo de caso prático, demonstramos como uma abordagem estruturada para o gerenciamento de riscos pode significativamente aumentar as taxas de sucesso de projetos de desenvolvimento de software.

1. Introdução

O desenvolvimento de software é inerentemente arriscado. Estudos mostram que uma porcentagem significativa de projetos de software não entrega o valor esperado, excede o orçamento ou não cumpre os prazos. A gestão de riscos sistemática tornou-se, portanto, uma disciplina crítica para o sucesso de projetos de TI.

1.1. Objetivos

Este trabalho tem como objetivos:

  • Identificar e classificar os principais riscos em projetos de software
  • Apresentar metodologias de análise e avaliação de riscos
  • Demonstrar estratégias de mitigação e resposta a riscos
  • Fornecer um estudo de caso prático de aplicação

1.2. Justificativa

De acordo com o CHAOS Report do Standish Group, apenas 31% dos projetos de software são bem-sucedidos. A má gestão de riscos é frequentemente citada como uma das principais causas de fracasso, justificando a necessidade de estudos aprofundados sobre o tema.

2. Revisão Teórica

Risco em projetos de software pode ser definido como um evento ou condição incerta que, se ocorrer, tem um efeito positivo ou negativo em pelo menos um objetivo do projeto.

2.1. Frameworks de Gerenciamento de Riscos

Os principais frameworks incluem:

  • PMBOK: Define processos de gerenciamento de riscos
  • CMMI: Fornece práticas para gerenciamento de riscos
  • ISO 31000: Norma internacional para gestão de riscos

3. Classificação dos Riscos em Projetos de Software

Categoria Exemplos de Riscos Impacto Potencial
Riscos Técnicos Tecnologia nova ou não testada, complexidade excessiva, requisitos de desempenho não realistas Atrasos, retrabalho, falha em atender especificações
Riscos de Gerenciamento Má definição de escopo, estimativas irrealistas, alocação inadequada de recursos Excesso de custos, atrasos, baixa qualidade
Riscos Organizacionais Mudanças de prioridades, conflitos internos, falta de comprometimento Descontinuidade do projeto, baixa moral da equipe
Riscos Externos Mudanças regulatórias, flutuações de mercado, problemas com fornecedores Necessidade de retrabalho, cancelamento do projeto

4.1. Identificação de Riscos

Técnicas comuns incluem brainstorming, método Delphi, entrevistas, análise SWOT e checklist baseado em experiências anteriores.

4.3. Planejamento de Respostas

As principais estratégias para riscos negativos:

  • Evitamento: Eliminar a ameaça
  • Transferência: Terceirizar o risco
  • Mitigação: Reduzir probabilidade ou impacto
  • Aceitação: Assumir o risco conscientemente

5. Estudo de Caso: Sistema de Gestão Acadêmica

5.1. Contexto do Projeto

Desenvolvimento de um sistema integrado de gestão acadêmica para uma universidade, com prazo de 12 meses e orçamento de R$ 500.000,00.

5.2. Riscos Identificados e Mitigação

Risco Probabilidade Impacto Estratégia de Mitigação
Mudanças frequentes de requisitos Alta Alto Implementar processo de controle de mudanças; usar metodologia ágil
Falta de competência técnica na equipe Média Alto Treinamento; contratação de especialistas; pair programming
Problemas de integração com sistemas legados Alta Médio Prototipação antecipada; testes de integração contínuos

5.3. Resultados

Com a aplicação sistemática de gestão de riscos, o projeto foi concluído com 10% de contingência de tempo e 15% de contingência orçamentária utilizada, considerado um sucesso pela instituição.

6. Ferramentas de Apoio

Diversas ferramentas podem auxiliar no gerenciamento de riscos:

  • JIRA: Para acompanhamento de issues e riscos
  • RiskyProject: Software especializado em análise de riscos
  • Planilhas personalizadas: Para projetos menores
  • Microsoft Project: Com funcionalidades básicas de gestão de riscos

7. Conclusões

A gestão de riscos é um componente essencial para o sucesso de projetos de software. Através da identificação precoce, análise sistemática e planejamento de respostas apropriadas, as organizações podem significantly aumentar suas taxas de sucesso.

Recomendações principais:

  • Integrar a gestão de riscos em todas as fases do projeto
  • Envolver todas as partes interessadas no processo de identificação de riscos
  • Manter um registro de riscos atualizado e acessível
  • Revisar regularmente os riscos e adaptar as estratégias de resposta
